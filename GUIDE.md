# Complete Run Guide — MRT Instruction Tuning Pipeline
# Lightning AI (open-source models only, no API key needed)
# ============================================================

## What every file does

| File | Purpose | Run it? |
|---|---|---|
| `config.py` | All settings — paths, column names, model names, hyperparameters | No — edit only |
| `data_utils.py` | Trip-tuple and persona formatting | No — imported by others |
| `check_data.py` | Validates your CSVs before anything else | Yes (Step 2) |
| `generate_instructions.py` | Loads Qwen2.5-7B, generates N instruction variants per task | Yes (Step 4) |
| `build_dataset.py` | Builds train/val/test JSON for each task | Yes (Step 5) |
| `lora_train_lib.py` | Shared QLoRA training logic | No — imported by train scripts |
| `train_lora_dest.py` | Fine-tunes on destination prediction | Yes (Step 6a) |
| `train_lora_ori.py` | Fine-tunes on origin prediction | Yes (Step 6b) |
| `train_lora_entry_time.py` | Fine-tunes on entry-time prediction | Yes (Step 6c) |
| `infer.py` | Runs predictions on the test set | Yes (Step 7) |
| `evaluate.py` | Computes accuracy metrics | Yes (Step 8) |

## What YOU provide (not generated by code)

```
dataset/
    trip_data.csv          ← your real MRT trip records
    persona_data.csv       ← your passenger-level statistics

prompt_store/gen/
    gen_pred_dest_instruct.txt       ← generation rules for dest task
    gen_pred_ori_instruct.txt        ← generation rules for ori task
    gen_pred_entry_time_instruct.txt ← generation rules for entry_time task

prompt_store/seed_task/
    seed_prompt_dest.txt             ← example instruction for dest
    seed_prompt_ori.txt              ← example instruction for ori
    seed_prompt_entry_time.txt       ← example instruction for entry_time
```

CSV column requirements:
- Both CSVs must have: `Card_Id` (the shared join key)
- trip_data.csv: `Card_Id, Gender, Age_On_June_2024, Entry_DT, Exit_DT, DayName, Entry_Station_Name, Exit_Station_Name`
- persona_data.csv: `Card_Id, Total_Trips, Active_Days, Unique_Origin, Unique_Dests, Avg_TT, Peak_Trips, Weekend_Trips`

If any column name differs, change the `COL_*` constants in `config.py`.

## Step-by-step commands

### STEP 0 — Install dependencies
```bash
pip install -r requirements.txt
```
Do this once. On Lightning AI the GPU environment usually has torch pre-installed,
but run this to make sure peft, bitsandbytes, and transformers versions are correct.

---

### STEP 1 — Place your CSV files
Copy your files to:
```
dataset/trip_data.csv
dataset/persona_data.csv
```
No command needed — just copy the files.

---

### STEP 2 — Validate your CSVs
```bash
python check_data.py
```
Look for `✅ No blocking issues found.`
Fix anything marked ❌ before continuing.
⚠️ warnings are informational only.

---

### STEP 3 — Author your prompt files (no command — use a text editor)

Create 6 plain-text files:

**prompt_store/gen/gen_pred_dest_instruct.txt** — generation rules, must end with:
```
List of 10 tasks:
```
(The number 10 controls how many instruction variants get generated.)

**prompt_store/seed_task/seed_prompt_dest.txt** — one complete example instruction:
```
<task definition>
You are predicting the exit station ...
________________________________________
<persona>
(Total_Trips, Active_Days, ...)
________________________________________
<data description>
...target_stay: (..., <next_station_name>)
________________________________________
<thinking guidance>
1. Scan history ...
```

Do the same for ori (mask Entry_Station_Name as `<entry_station_name>`,
Exit_DT and Exit_Station_Name become None in target_stay) and entry_time
(mask Entry_DT as `<next_entry_time>`).

---

### STEP 4 — Generate diverse instructions using Qwen2.5-7B-Instruct
```bash
# Generate for all 3 tasks (downloads model ~15GB on first run):
python generate_instructions.py

# Or one task at a time:
python generate_instructions.py --task dest
python generate_instructions.py --task ori
python generate_instructions.py --task entry_time

# Regenerate if you changed your prompt files:
python generate_instructions.py --force
```

**What happens:** Qwen2.5-7B-Instruct is loaded on GPU, it reads your gen rules
+ seed prompt, generates N instruction variants, saves them as:
```
prompt_store/instruction/dest/dest_instruction_01.txt
prompt_store/instruction/dest/dest_instruction_02.txt
... (N files)
prompt_store/instruction/dest/dest_instructions_raw.txt  ← full model response
```
After saving, GPU memory is freed automatically.

**This step only runs once** — results are cached. Re-running without `--force`
prints "Skipping" if files already exist.

---

### STEP 5 — Build the instruction-tuning datasets
```bash
# Build all 3 tasks:
python build_dataset.py

# Or one task at a time:
python build_dataset.py --task dest
```

**What happens:**
- Joins trip_data.csv + persona_data.csv on Card_Id
- Drops passengers with fewer than 8 trips (change `MIN_TRIPS_PER_PASSENGER` in config.py)
- Splits passengers 80/10/10 into train/val/test (at the passenger level, not trip level)
- Assigns one random instruction style per passenger per task
- Slides a window over each passenger's trip sequence to build (history, context, target) samples
- Saves 9 JSON files:

```
data4FT/ori_train.json      data4FT/ori_val.json      data4FT/ori_test.json
data4FT/dest_train.json     data4FT/dest_val.json     data4FT/dest_test.json
data4FT/entry_time_train.json  ...
data4FT/passenger_split.json   ← which Card_Id went to which split
```

**TIP:** With 590K passengers, this produces a very large dataset.
For a fast first run, set `MAX_PASSENGERS = 20000` in config.py first.
After confirming results look good, set it back to `None` for full training.

---

### STEP 6 — Train (one script per task, run sequentially)

```bash
# 6a — Destination prediction model:
python train_lora_dest.py

# 6b — Origin prediction model:
python train_lora_ori.py

# 6c — Entry time prediction model:
python train_lora_entry_time.py
```

**Useful flags (same for all 3 scripts):**
```bash
--no_qlora          # plain bf16 LoRA instead of 4-bit QLoRA (needs more VRAM)
--epochs 5          # override number of epochs (default: 3)
--batch_size 4      # override batch size (default: 2)
--lr 5e-5           # override learning rate (default: 1e-4)
```

**What gets saved:**
```
modelSave/llama3_1_mrt_qlora_dest/checkpoint-100/
modelSave/llama3_1_mrt_qlora_dest/checkpoint-200/
...
modelSave/llama3_1_mrt_qlora_dest/final_adapter/   ← USE THIS for inference
modelSave/llama3_1_mrt_qlora_ori/final_adapter/
modelSave/llama3_1_mrt_qlora_entry_time/final_adapter/
```

**VRAM guide (Lightning AI A100 80GB):**
- Default settings (4-bit QLoRA, batch_size=2): ~12GB VRAM per task
- If you hit OOM: try `--batch_size 1 --grad_accum 8`
- Each task trains independently — you can run them back-to-back or in parallel on separate machines

**Note:** The base model (Llama 3.1 8B, ~16GB) downloads from HuggingFace on first run.
You may need to accept the license at huggingface.co/meta-llama first.

---

### STEP 7 — Run inference on the test set
```bash
# Always do a smoke test first (one example, confirms the model works):
python infer.py --task dest --smoke_test --use_4bit

# Full test set inference:
python infer.py --task dest --use_4bit
python infer.py --task ori --use_4bit
python infer.py --task entry_time --use_4bit

# Run only the first 100 samples (quick check):
python infer.py --task dest --use_4bit --limit 100

# Raw base model (no adapter) for baseline comparison:
python infer.py --task dest --no_adapter --use_4bit
```

**Output CSV saved to:**
```
output/dest_predictions.csv
output/ori_predictions.csv
output/entry_time_predictions.csv
```

**Columns in the CSV:**
- `index` — sample number
- `raw_output` — full text the model generated (including chain-of-thought)
- `prediction` — extracted predicted value
- `ground_truth` — correct answer from the test set
- `parse_error` — "PARSE_FAILURE" if the output wasn't parseable JSON; None if OK

---

### STEP 8 — Evaluate
```bash
# Station prediction (exact match):
python evaluate.py --predictions_csv output/dest_predictions.csv --task dest
python evaluate.py --predictions_csv output/ori_predictions.csv  --task ori

# Entry time (exact match + within-N-minutes tolerance):
python evaluate.py --predictions_csv output/entry_time_predictions.csv \
    --task entry_time --time_tolerance_min 15
```

**Output metrics:**
```
=== Exact-match metrics ===
  n_total: 1800
  n_parsed: 1760              ← how many were parseable JSON
  exact_match_acc: 0.623      ← 62.3% correct
  parse_failure_rate: 0.022   ← 2.2% of outputs couldn't be parsed

=== Time-tolerance metrics (+/- 15 min) ===   [entry_time only]
  within_tolerance_acc: 0.71  ← 71% within 15 minutes
  mean_abs_diff_min: 8.4
  median_abs_diff_min: 5.0
```

## Common issues

| Problem | Fix |
|---|---|
| `Card_Id column not found` | Check your CSV column name and update `COL_PASSENGER_ID` in config.py |
| `Exit station column not found` | Your CSV may use `Exit_Satation_Name` (typo) — it is auto-detected |
| `FileNotFoundError: gen rules not found` | You haven't authored prompt_store/gen files yet (Step 3) |
| `FileNotFoundError: instruction files not found` | Run generate_instructions.py first (Step 4) |
| `FileNotFoundError: dest_train.json not found` | Run build_dataset.py first (Step 5) |
| CUDA out of memory during training | Add `--batch_size 1 --grad_accum 8` to the train command |
| High parse_failure_rate (>5%) | Add "output ONLY the JSON, no other text" to your seed prompt's thinking guidance |
| Model not downloading | Accept the Llama 3.1 license at huggingface.co/meta-llama then run `huggingface-cli login` |
